# 中医药高质量数据集构建演示系统设计说明

## 目标

构建一个面向中医药高质量数据集建设的前端演示系统。系统用于展示从多源中医药数据接入，到数据采集、数据标注，再到形成数据集产品的全过程，并提供数据集超市页面展示最终产出的数据产品。

项目新建为 Vite + Vue3 应用，核心页面包括：首页、数据集构建页、数据集超市列表页、数据集详情页。首页已有明确方向，本设计仅保留入口和路由边界，后续按用户具体要求实现。

## 总体页面结构

路由规划：

- `/`：首页。作为系统入口，后续按用户明确要求实现视觉和内容。
- `/build`：数据集构建页。系统核心页面，展示中医药数据集构建流程大屏。
- `/market`：数据集超市列表页。展示已形成的数据集产品，支持筛选、搜索、排序和跳转详情。
- `/market/:id`：数据集详情页。展示数据集摘要、介绍、元数据、说明文档、供应方和相关推荐。

整体设计采用“沉浸大屏 + 产品货架”的组合：

- 构建页负责制造系统记忆点，强调中医药数据从来源到产品的生成过程。
- 超市页负责承接构建结果，展示可检索、可查看、可下载的数据集资产。
- 假接口层统一提供页面数据，模拟真实系统的数据加载、列表、详情和相关推荐。

## 数据集构建页

构建页采用半交互型流程大屏。页面进入后自动播放构建流程，用户也可以点击阶段节点查看对应详情。

### 视觉方向

主视觉为 Three.js 三维数据星图：

- 中心星核表示“中医药高质量数据集”。
- 四个阶段节点围绕中心分布，并通过粒子连线与中心关联。
- 节点按流程依次高亮：数据源接入、数据采集、数据标注、产品形成。
- 粒子流沿连线汇入中心，表现数据被治理、加工和沉淀为产品。
- 场景保持轻微自动旋转，形成沉浸式大屏效果。

中医药元素通过业务语义和轻量视觉细节表达：

- 顶部或背景中加入脉象波形线，表现中医诊疗语义。
- 背景可使用低透明度本草纹理、经络式曲线或中医药关键词浮层。
- 色彩以深蓝、青绿为科技底色，少量药金作为高亮点缀。
- 避免做成古风页面，整体仍是现代科技大屏。

### 流程阶段

1. 数据源接入
   - 数据来源包括古籍文献、临床病历、方剂本草、科研语料、药材图像、舌象图像。
   - 信息面板展示接入来源数、数据类型、接入方式和数据合规状态。

2. 数据采集
   - 采集能力包括 OCR 识别、接口同步、批量导入、术语抽取、结构化入库。
   - 信息面板展示采集量、结构化率、术语抽取数量和异常数据比例。

3. 数据标注
   - 标注对象包括疾病、证候、方剂、中药、功效、主治、疗效关系。
   - 信息面板展示标注任务数、专家复核进度、一致性评分和质检层级。

4. 产品形成
   - 形成专病数据集、方剂知识集、中医药评测集、舌象图像数据集和知识图谱资产。
   - 信息面板展示数据集数量、质量评分、可下载资产数和最近更新时间。

### 交互

- 默认自动播放流程。
- 点击任意阶段节点后，暂停自动播放并切换右侧阶段详情。
- 点击播放按钮恢复自动演示。
- 底部提供阶段切换控制，支持按顺序查看四个阶段。
- 顶部指标条展示总数据源、标准化率、专家质检层级、已形成数据集数等关键指标。

### 降级

如果 Three.js 初始化失败，页面展示 CSS 静态星图兜底。兜底图仍包含中心数据集核心、四个阶段节点、连线和阶段信息面板，保证演示页面可用。

## 数据集超市列表页

列表页布局贴近用户提供的参考图：左侧导航目录，右侧主列表。

### 左侧导航目录

按中医药数据类型组织筛选项：

- 中医文本：古籍整理、方剂抽取、证候分类、医案问答。
- 医学影像：舌象识别、面诊图像、药材图像。
- 临床数据：专病队列、疗效评估、随访记录。
- 知识资产：知识图谱、术语词表、评测集。

筛选项以标签按钮形式展示，点击后刷新列表数据。筛选状态由前端维护，数据由假接口返回。

### 右侧主区域

主区域包含：

- 面包屑：`首页 > 数据集专区`。
- Tab：`数据集列表`、`数据集建设意向列表`。
- 搜索框：输入关键词搜索中医药数据集。
- 搜索按钮和发布数据集按钮。
- 高级搜索入口和综合排序入口。
- 数据集卡片列表。

数据集卡片展示：

- 数据集标识或默认文字 Logo。
- 数据集名称。
- 简介摘要。
- 分类、模态、行业、领域标签。
- 供应方、下载量、数据大小、更新时间。
- 收藏按钮。

示例数据集包括：

- 中医古籍方剂-证候关系语料集。
- 舌象图像分类与体质辨识数据集。
- 中医医案问答与经验分享语料集。
- 中药材图像识别数据集。
- 专病疗效评估随访数据集。

## 数据集详情页

详情页延续用户参考图结构：顶部摘要卡片，左侧主内容，右侧供应方和相关推荐。

### 顶部摘要卡片

摘要卡片与列表卡片信息一致，展示：

- 数据集 Logo。
- 数据集名称。
- 简介。
- 分类、模态、行业、领域标签。
- 供应方、下载量、大小、更新时间。
- 收藏按钮。

### 主内容区

主内容区包含三个 Tab：

- 数据集介绍。
- 数据样例。
- 元数据。

数据集介绍包含：

- 数据集概述。
- 适用任务。
- 数据来源。
- 标注规则。
- 质量控制说明。
- 使用限制。

元数据以表格形式展示：

- 数据集唯一标识。
- 数据集分类。
- 数据模态。
- 数据来源。
- 数据规模。
- 标注体系。
- 质检方式。
- 是否涉及个人信息。
- 是否需要授权使用。
- 更新频率。

说明文档区域展示数据说明书预览，并提供下载按钮。

### 右侧栏

右侧栏包含：

- 供应方介绍：机构名称、能力描述、联系方式展示区。
- 相关数据：推荐同类型或同领域的数据集。

## 假接口与数据模型

项目使用前端内部假接口层模拟真实接口，建议放在 `src/services/mockApi.js`。

假接口提供：

- 获取构建流程阶段数据。
- 获取构建页指标数据。
- 获取数据集分类筛选项。
- 获取数据集列表。
- 获取数据集详情。
- 获取相关推荐。
- 模拟收藏状态切换。
- 模拟下载动作状态。

核心数据模型：

```js
{
  id: 'formula-syndrome-corpus',
  name: '中医古籍方剂-证候关系语料集',
  summary: '面向方剂知识抽取与证候分类任务，包含古籍条文、方药组成、功效主治与证候关系标注。',
  category: '中医文本',
  taskType: '关系抽取',
  modality: '文本/关系抽取',
  industry: '中医药',
  domain: '科学研究',
  provider: '中医药数据中心',
  downloads: 0,
  size: '224MB',
  updatedAt: '2026-06-15',
  tags: ['古籍整理', '方剂抽取', '证候分类'],
  metadata: {},
  document: {},
  relatedIds: []
}
```

## 组件边界

构建页组件：

- `BuildScene`：Three.js 场景容器，负责星核、节点、连线、粒子和动画。
- `StageInfoPanel`：展示当前阶段说明、能力列表和指标。
- `BuildMetrics`：顶部关键指标条。
- `FlowControls`：底部播放、暂停和阶段切换控制。

超市列表页组件：

- `FilterSidebar`：左侧筛选目录。
- `DatasetSearchBar`：搜索、发布、高级搜索和排序入口。
- `DatasetCard`：数据集列表卡片。

详情页组件：

- `DatasetSummary`：顶部摘要卡片。
- `DatasetTabs`：介绍、样例、元数据切换。
- `MetadataTable`：元数据表格。
- `ProviderPanel`：供应方介绍。
- `RelatedDatasets`：相关推荐。

通用组件：

- `AppHeader`：站点导航。
- `Breadcrumb`：面包屑。
- `TagBadge`：标签样式。
- `EmptyState`：空状态。
- `LoadingState`：加载状态。

## 技术选型

- 构建工具：Vite。
- 前端框架：Vue3。
- 路由：Vue Router。
- 3D 场景：Three.js。
- 样式：普通 CSS 或组件内 scoped CSS，优先保持项目轻量。
- 图标：如需使用图标，采用 lucide-vue-next。
- 数据：前端假接口层，不接真实后端。

## 响应式与可访问性

- 桌面端优先展示完整大屏效果。
- 移动端构建页降低 3D 画面占比，阶段面板改为下方堆叠。
- 超市页移动端左侧筛选改为顶部折叠筛选区。
- 所有按钮提供可读文字或标题。
- 避免文本遮挡、按钮文字溢出和卡片内容重叠。

## 验证计划

实现后需要验证：

- 应用可以正常启动。
- 首页、构建页、超市列表页、详情页路由正常。
- 构建页 Three.js 画面非空，中心星核、阶段节点和连线可见。
- 构建页自动播放流程有效，点击节点能切换阶段详情。
- Three.js 失败时静态兜底画面可见。
- 超市页搜索、筛选、排序入口和详情跳转正常。
- 详情页摘要、介绍、元数据、说明文档、供应方和相关推荐正常展示。
- 桌面和移动端主要页面无明显重叠、溢出或空白异常。

## 非目标

- 不实现真实后端、数据库、权限系统或文件上传。
- 不在本阶段设计首页具体视觉和文案。
- 不实现真实数据下载，仅模拟下载状态。
- 不实现复杂数据集发布流程，仅保留发布入口。
