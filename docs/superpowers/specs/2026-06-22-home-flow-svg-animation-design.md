# 首页数据流 SVG 动画设计

## 目标

将首页“中医药（苗药）数据集构建流程”区域的静态流程图升级为动画展示，同时保持原图的构图、元素、文字、主题和配色不变。动画需要清晰表达数据从左侧数据源进入中间处理环节，最终流向右侧应用场景的过程。

## 实现方式

新增独立组件 `HomeFlowAnimation.vue`。组件使用响应式 SVG 作为容器：

- SVG 底层通过 `<image>` 完整嵌入现有 `/images/index-00.png`。
- SVG 上层叠加透明动画路径、流动粒子、节点光晕和局部扫光。
- 首页仅将当前静态 `<img>` 替换为该组件，不改变标题、产品区或数据加载逻辑。

这种方式能够最大限度保持原图视觉一致，同时让动画代码与页面其他逻辑隔离。

## 动画阶段

### 1. 数据源接入

- 在原图左侧文本、PDF、图像、网页和命令行数据源对应线路上叠加低透明度流动粒子。
- 粒子沿原图虚线路径向中间移动，不新增新的可见线路。
- 不同来源使用轻微错峰，避免所有粒子同时出现。

### 2. 数据处理

- 数据到达中间处理面板后，按从左到右顺序触发节点脉冲。
- 数据提取、数据清洗、数据标注、质量评估等区域依次出现柔和扫光。
- 扫光颜色取自原图青色、蓝紫色和药金色，不改变底图本身。

### 3. 应用输出

- 粒子从中间处理区沿原图输出线路向右侧分流。
- 大模型预训练、诊疗、微调、产业应用、评测和古籍检索等终点依次出现呼吸光晕。
- 一轮完成后短暂停顿并自动循环。

## 视觉约束

- 原图必须完整保留，不重绘、不裁剪主要元素。
- 动画覆盖层应克制，不能遮挡原图文字。
- 页面配色继续使用草本青、宣纸白、青蓝和少量药金色。
- 外框保持 8px 以内圆角，与当前页面一致。
- 桌面端保持原图横向比例；小屏允许等比缩放，不拉伸变形。

## 动画与性能

- 优先使用 SVG `animateMotion`、CSS `stroke-dashoffset` 和 `opacity/transform` 动画。
- 避免持续修改布局属性，动画仅作用于透明度和变换。
- 使用 `prefers-reduced-motion`：系统开启减少动态效果时，停止粒子移动，仅保留静态底图。
- SVG 动画区域标记为装饰性，不影响页面语义和键盘操作。

## 组件边界

`HomeFlowAnimation.vue` 只负责：

- 渲染底图。
- 管理 SVG 动画覆盖层。
- 响应容器尺寸和减少动态效果设置。

`HomeView.vue` 继续负责首页结构、数据集分类和数据加载，不承载动画细节。

## 验收标准

- 首页流程图与当前原图在静止状态下视觉一致。
- 数据流能明确表现“左侧数据源 → 中间治理处理 → 右侧应用场景”。
- 动画循环连贯，没有明显跳帧、遮字或溢出。
- SVG 在常见桌面和移动宽度下等比缩放。
- 开启减少动态效果后不再播放持续动画。
